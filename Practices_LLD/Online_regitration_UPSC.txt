public class ExamSystem {
    public static void main(String[] args) {

        // Base prototype
        UPSCApplication baseForm =
            new UPSCApplication("India", "General Studies", "Offline", 2024);

        // Clone for Candidate 1
        UPSCApplication c1 = (UPSCApplication) baseForm.clone();
        c1.candidateName = "Gopi";
        c1.rollNumber = "UPSC001";

        // Clone for Candidate 2
        UPSCApplication c2 = (UPSCApplication) baseForm.clone();
        c2.candidateName = "Adithya";
        c2.rollNumber = "UPSC002";

        c1.print();
        c2.print();
    }
}

// Prototype interface
interface ExamApplication {
    ExamApplication clone();
}

// Concrete Prototype
class UPSCApplication implements ExamApplication {

    String country;
    String examType;
    String mode;
    int year;

    String candidateName;
    String rollNumber;

    UPSCApplication(String country, String examType, String mode, int year) {
        this.country = country;
        this.examType = examType;
        this.mode = mode;
        this.year = year;
    }

    @Override
    public ExamApplication clone() {
        return new UPSCApplication(this.country, this.examType, this.mode, this.year);
    }

    void print() {
        System.out.println(
            candidateName + " | " + rollNumber + " | " +
            examType + " | " + year + " | " + mode
        );
    }
}

‚≠ê How to explain in interview (2 lines)

In an online exam system, most application data remains the same for all candidates. Using the Prototype Pattern, we clone a base application object and customize only candidate-specific fields, improving performance and reducing object creation complexity.
